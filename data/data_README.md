## Raw Data (`raw/`)

### ITACA Archives

**Files:**
- `query_acc.zip` - Acceleration recordings (.ASC format)
- `query_vel.zip` - Velocity recordings (.ASC format)
- `query_disp.zip` - Displacement recordings (.ASC format)

**Format:** ASCII text files, one per station-component pair

**Contents:**
- Header: metadata (station, event, coordinates, PGA, etc.)
- Body: time series at 200 Hz sampling rate
- Components: HNE (East), HNN (North), HNZ (Vertical)

**Source:** ITACA database (Istituto Nazionale di Geofisica e Vulcanologia)

**Processing:** See notebooks `01a_metadata.ipynb` and `01b_signals.ipynb`

## Processed Data (`processed/`)

All outputs organized by analysis stage (01‚Äì05) and signal type (acceleration, velocity, displacement).

---

### 01a. Metadata (`01a_metadata/`)

Cleaned and enriched metadata tables.

**Structure:** `01a_metadata/{signal_type}/`

**Files per signal type:**
- `metadata_clean_{signal_type}.parquet` - Cleaned metadata
- `metadata_variable_{signal_type}.parquet` - Variable statistics

**Columns (clean):**
- Station: CODE, LATITUDE, LONGITUDE
- Event: LATITUDE, LONGITUDE, DEPTH_KM, MAGNITUDE
- Picks: TIME_P_ARRIVAL, TIME_S_ARRIVAL (theoretical)
- Ground motion: PGA, PGV, PGD, TIME_PGA, etc.
- Distance: EPICENTRAL_DISTANCE_KM, HYPOCENTRAL_DISTANCE_KM
- Crustal: V_P_CRUST, V_S_CRUST

**Generated by:** `01a_metadata.ipynb`

---

### 01b. Preprocessed Signals (`01b_signals/`)

Cleaned time series ready for analysis.

**Structure:** `01b_signals/{signal_type}/`

**Files per signal type:**
- `{signal_type}_preprocessed_pdf.parquet` - For PDF analysis (normalized)
- `{signal_type}_preprocessed_scaling.parquet` - For moment scaling (unnormalized)

**Columns:**
- `file` (str) - File identifier
- `sample` (int) - Sample index
- `{signal_type}` (float) - Signal value in physical units
- `{signal_type}_normalized` (float) - Standardized signal (PDF files only)

**Physical units:**
- Acceleration: cm/s¬≤
- Velocity: cm/s
- Displacement: cm

**Generated by:** `01b_signals.ipynb`

---

### 02. PDF Analysis (`02_pdf_analysis/`)

Probability distribution characterization.

**Structure:** `02_pdf_analysis/{signal_type}/`

**Files per signal type:**
- `gaussian_fit_results.parquet` - Gaussian fit parameters and Anderson-Darling test
- `heavy_tail_results.parquet` - Tail exponent estimation (L√©vy, Student-t, power-law)

**Columns (Gaussian fit):**
- `file`, `station`, `stream` - Identifiers
- `mu`, `std` - Fit parameters
- `kurtosis`, `skewness` - Distribution shape
- `ad_statistic`, `ad_critical_5pct` - Anderson-Darling test
- `non_gaussian` (bool) - Null hypothesis rejected

**Columns (heavy tail):**
- `file`, `station`, `stream` - Identifiers
- `aic_gaussian`, `aic_laplace`, `aic_student_t`, `aic_levy_stable` - Model comparison
- `best_fit_aic` (str) - Best-fitting distribution
- `student_t_df` (float) - Degrees of freedom (lower = heavier tails)
- `levy_alpha` (float) - Stability parameter (Œ± ‚àà (0,2], Œ±=2 is Gaussian)
- `power_law_exp` (float) - Hill estimator tail exponent

**Generated by:** `02_pdf_analysis.ipynb`

---

### 03a. AR-AIC Phase Detection (`03a_phase_identification_ar_pick/`)

Autoregressive-AIC onset detection results.

**Structure:** `03a_phase_identification_ar_pick/{signal_type}/`

**Files per signal type:**
- `df_full_{signal_type}_ar_pick.parquet` - Complete metadata with detected onsets
- `signals_{signal_type}_dict.pkl` - Dictionary of signal arrays
- `windowed_{signal_type}_{coda_method}_ar_pick.pkl` - Segmented signals (4 methods)

**Coda methods:** `rautian`, `arias`, `envelope`, `median`

**Columns (df_full):**
- All metadata columns
- `t_p_detected_samples`, `t_p_detected_seconds` - P-wave onset
- `t_s_detected_samples`, `t_s_detected_seconds` - S-wave onset
- `t_coda_{method}_samples`, `t_coda_{method}_seconds` - Coda onset per method

**Windowed structure:**
```python
{
  'station_code': {
    'component': {
      'pre_event': {'signal': array, 'time': array, 'start_samples': int, ...},
      'p_wave': {...},
      's_wave': {...},
      'coda': {...}
    }
  }
}
```

**Generated by:** `03_phase_identification.ipynb`

---

### 03b. PhaseNet Phase Detection (`03b_phase_identification_phasenet/`)

Deep-learning phase picker results.

**Structure:** `03b_phase_identification_phasenet/{signal_type}/`

**Files per signal type:**
- `df_full_{signal_type}_phasenet.parquet` - Complete metadata with PhaseNet picks
- `phasenet_picks_{signal_type}.csv` - Raw PhaseNet output
- `signals_{signal_type}_dict.pkl` - Dictionary of signal arrays
- `windowed_{signal_type}_{coda_method}_phasenet.pkl` - Segmented signals (4 methods)

**Additional file:** `phasenet_picks_instance_corrected.csv` (coordinate-corrected picks)

**Generated by:** `03_phase_identification.ipynb`

---

### 04a. Spatial Ensemble Scaling (`04a_moment_scaling_spatial/`)

Moment scaling with spatial averaging across stations.

**Structure:** `04a_moment_scaling_spatial/{picker}/{signal_type}/{coda_method}/`

**Hierarchy:**
- **Picker:** `ar_pick`, `phasenet`
- **Signal type:** `acceleration`, `velocity`, `displacement`
- **Coda method:** `rautian`, `arias`, `envelope`, `median`

**Files per leaf directory:**
- `ensemble_spatial_summary.parquet` - Scaling exponents Œ∂(q) for all windows
- `ensemble_spatial_moments_{window}.parquet` - Detailed moment curves (4 files)

**Windows:** `pre_event`, `p_wave`, `s_wave`, `coda`

**Columns (summary):**
- `window` (str) - Window name
- `q` (float) - Moment order
- `zeta` (float) - Scaling exponent
- `zeta_err` (float) - Standard error from fit
- `r_squared` (float) - Goodness of fit
- `n_signals` (int) - Ensemble size
- `tau_min`, `tau_max` (float) - Time lag range (seconds)

**Columns (moments):**
- `tau` (float) - Time lag (seconds)
- `tau_samples` (int) - Time lag (sample indices)
- `q` (float) - Moment order
- `moment_mean` (float) - Ensemble-averaged M_q(œÑ)
- `moment_std` (float) - Standard deviation across ensemble
- `n_signals` (int) - Number of signals averaged

**Total files:** 2 pickers √ó 3 signal types √ó 4 coda methods √ó 5 files = **120 files**

**Generated by:** `04a_moment_scaling_spatial.ipynb`

---

### 04b. Temporal Ensemble Scaling (`04b_moment_scaling_temporal/`)

Time-averaged moment scaling within single signals (sliding window).

**Structure:** `04b_moment_scaling_temporal/{signal_type}/{coda_method}/`

**Status:** Currently contains acceleration/rautian only

**Files:**
- `ensemble_spatial_summary.parquet` - Temporal ensemble results

**Note:** This analysis is **work in progress** ‚Äî only partial results available.

**Generated by:** `04b_moment_scaling_temporal.ipynb`

---

### 05. Sensitivity Analysis (`05_sensitivity_analysis/`)

Monte Carlo uncertainty quantification for phase pick perturbations.

**Structure:** `05_sensitivity_analysis/phasenet/{signal_type}/`

**Main files:**
- `sensitivity_results_phasenet_complete.pkl` - Complete results object
- `sensitivity_summary_phasenet.csv` - Summary statistics

**Intermediate results:** `intermediate/`
Contains checkpoint files for resumable computation:
- `mc_{signal_type}_{coda}_run{N}.pkl` - Monte Carlo iteration N
- `results_{signal_type}_{coda}_{perturbation_type}.pkl` - Per-perturbation results
- `complete_{signal_type}_{coda}.pkl` - Complete per-configuration results

**Perturbation types:**
- `noise_small`, `noise_medium`, `noise_large` - Gaussian noise (œÉ = 0.05, 0.10, 0.20s)
- `bias_early`, `bias_late` - Systematic bias (¬±0.15s)

**Total intermediate files:** 3 signal types √ó 4 coda methods √ó (5 iterations + 5 perturbations + 1 complete) = **132 files**

**Generated by:** `05_sensitivity_analysis.ipynb`

## File Formats

### Parquet (.parquet)
Apache Parquet columnar format for efficient storage and fast queries.

**Reading:**
```python
import pandas as pd

# Full read
df = pd.read_parquet('data/processed/01a_metadata/acceleration/metadata_clean_acc.parquet')

# Column subset
df = pd.read_parquet('file.parquet', columns=['station', 'q', 'zeta'])

# Filtered read
df = pd.read_parquet('file.parquet', filters=[('q', '==', 2.0)])
```

---

### Pickle (.pkl)
Python pickle format for complex nested structures (dictionaries, nested arrays).

**Reading:**
```python
import pickle

with open('signals_acceleration_dict.pkl', 'rb') as f:
    signals = pickle.load(f)
```

**Warning:** Pickle files are Python-version and platform-dependent. Use for intermediate storage only.

Comma-separated values for lightweight tabular data.

**Reading:**

df = pd.read_csv('phasenet_picks_acceleration.csv')

## Naming Conventions

### Signal Types
- `acc` / `acceleration` - Acceleration (cm/s¬≤)
- `vel` / `velocity` - Velocity (cm/s)
- `dis` / `disp` / `displacement` - Displacement (cm)

### Phase Pickers
- `ar_pick` - AR-AIC method
- `phasenet` - PhaseNet deep learning

### Coda Detection Methods
- `rautian` - Theoretical (2√ót_S from origin)
- `arias` - Arias intensity D5-95
- `envelope` - Amplitude-based threshold
- `median` - Robust combination of all three

### Time Representation
Dual format to avoid rounding errors:
- `*_samples` (int) - Sample indices (primary for computation)
- `*_seconds` (float) - Time in seconds (for visualization/output)

## Notes

**CRUST1.0 Model:**
- Grid resolution: 1¬∞ √ó 1¬∞
- Layer structure: 9 layers from sediment to upper mantle
- Used for theoretical arrival time computation
- Not modified ‚Äî reference copy from original distribution

**Dual Time Representation:**
- Sample indices used for computation (avoids rounding)
- Seconds used for output/visualization
- Conversion: `seconds = samples / sampling_rate` (200 Hz)

**Windowed Signal Storage:**
- Pickle format for nested dictionary structure
- Each window contains: signal, time, boundaries (samples + seconds), duration
- Four non-overlapping windows: pre-event, P-wave, S-wave, coda

**Sensitivity Analysis:**
- 100 Monte Carlo iterations per configuration
- Intermediate checkpoints every 20 iterations
- Results stored per signal type, coda method, and perturbation type
- Resumable: can continue from last checkpoint if interrupted
